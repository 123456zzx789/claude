# 直播并发控制系统 — 测试说明

## 1. 测试概述

测试目标为验证 LiveStreamManager 并发控制核心逻辑的正确性。测试框架采用 MSTest 3.6.4。开发方法采用 TDD（测试驱动开发）：先编写测试用例，在 LiveStreamManager 类尚未实现时测试编译失败（Red），随后实现 TryStart、Stop、StopAll 等核心方法使测试通过（Green），最后将 Form1 中散落的并发状态字段统一抽入 LiveStreamManager，代码从 233 行精简至 208 行（Refactor）。

## 2. 测试用例与结果

共设计 4 个测试用例，全部通过，总耗时约 4 秒。

TC-01（TryStart_超过并发上限_应返回false）：将并发上限设为 3，依次启动 3 路流均成功，尝试启动第 4 路时 TryStart 返回 false，验证并发上限控制有效。TC-02（Stop_释放槽位后_可重新启动）：占满 3 个槽位后停止其中 1 路，槽位释放后新的流可以成功启动，验证槽位释放与重新分配逻辑正确。TC-03（TryStart_重复启动同一流_应返回false）：对同一房间 ID 调用两次 TryStart，第一次成功返回 true，第二次被拒绝返回 false，验证重复启动防护机制有效。TC-04（StopAll_应停止所有流_活跃数归零）：3 路流同时运行中调用 StopAll，全部停止后 ActiveCount 归零、AvailableSlots 恢复为上限值，验证批量停止功能正确。

## 3. 代码覆盖

测试覆盖 LiveStreamManager 全部公开方法。TryStart 覆盖三种场景：正常启动、并发上限拒绝、重复启动防护。Stop 覆盖运行中停止与槽位释放。StopAll 覆盖批量停止与状态归零。IsRunning 由 TC-03 隐式覆盖。ActiveCount 和 AvailableSlots 属性由全部测试用例的断言覆盖。
