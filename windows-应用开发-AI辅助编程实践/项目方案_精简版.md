# 直播并发控制系统 — 项目方案

## 1. 项目背景

随着直播行业的快速发展，运营人员常常需要同时监控多个直播间的实时画面。传统做法是打开多个浏览器窗口分别查看，窗口切换繁琐且消耗大量系统资源。

本项目基于 C# WinForms + WebView2 构建轻量级直播并发控制系统，在单一窗口中嵌入多路直播画面，通过信号量机制限制同时播放的直播流数量，防止系统资源耗尽。

## 2. 功能需求

系统提供 3 个直播间面板，每个面板包含 WebView2 播放区域、频道下拉选择框和独立启停按钮，用户可对每个房间独立控制开播与停播。通过 LiveStreamManager 内置的信号量计数机制，系统将同时播放的直播流上限限制为 3 路，超出上限时拒绝启动新流。频道选择框内置 6 个 CCTV 频道选项，切换频道时自动停止旧流并重启新流。每个 WebView2 在页面加载完成后自动注入 CSS 脚本，隐藏导航栏、弹幕、侧栏和广告等非视频区域，仅保留 video 元素全屏铺满。WebView2 ZoomFactor 设为 0.22 适配小窗口展示。底部状态栏实时显示当前并发状态、活跃直播数和可用槽位数。

## 3. 技术选型

项目基于 .NET Framework 4.7.2，使用 Windows Forms 构建桌面界面，符合课程核心技术栈要求。直播画面嵌入采用 Microsoft.Web.WebView2（版本 1.0.2903.40），基于 Edge Chromium 内核，替代已废弃的 WebBrowser 控件，支持现代 Web 页面渲染。测试框架选用 MSTest 3.6.4，与 Visual Studio 深度集成。未选用 WPF（学习成本高，超出课程要求），未选用 VLC 或 ffmpeg 直接拉流（各直播平台无公开 RTMP/RTSP 地址），未选用 Electron（不符合课程 Windows 程序设计主题且内存占用大）。

## 4. 系统架构

项目采用两层架构。UI 层为 Form1 窗体，包含三个 GroupBox 划分的直播间区域，每个区域内的 Panel 承载 WebView2 控件，配合 ComboBox 频道选择和 Button 启停控制。业务层为独立的 LiveStreamManager 类，提供 TryStart、Stop、StopAll、IsRunning 等接口，以及 ActiveCount 和 AvailableSlots 属性。UI 层通过调用 LiveStreamManager 的接口来完成并发控制，LiveStreamManager 不依赖任何 UI 控件，可被单元测试完全隔离测试。

并发控制流程：用户点击开播时，TryStart 执行两层检查——先查字典判断同一房间 ID 是否已在运行中（防重复启动），再比较当前活跃数与上限（防超并发）。检查通过后分配槽位并递增计数，WebView2 导航至对应频道 URL，注入 CSS 净化页面，随后通过 CancellationToken 进入无限等待状态。用户点击停播时触发 CancellationToken.Cancel()，抛出 OperationCanceledException，finally 块确保导航至 about:blank 并释放槽位。

## 5. 项目结构

解决方案 实验4.sln 下包含两个项目。主项目 实验4 包含 Form1.cs（UI 事件与 WebView2 控制逻辑）、Form1.Designer.cs（设计器生成的控件布局代码）、LiveStreamManager.cs（并发管理器，核心业务逻辑）、Program.cs（应用入口）。单元测试项目 实验4.Tests 包含 LiveStreamManagerTests.cs，其中定义了 4 个测试用例，覆盖并发上限拒绝、槽位释放与重新分配、重复启动防护、批量停止等核心场景。

## 6. 开发环境

操作系统为 Windows 11 Pro，IDE 为 Visual Studio 2022，目标框架 .NET Framework 4.7.2，测试框架 MSTest 3.6.4，NuGet 依赖 Microsoft.Web.WebView2 1.0.2903.40，版本控制使用 Git。
