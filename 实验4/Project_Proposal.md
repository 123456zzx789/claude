# 直播并发控制系统 — 项目方案

## 1. 项目背景

随着直播行业的快速发展，运营人员常常需要同时监控多个直播间的实时画面，例如电竞赛事多路信号、多平台直播间内容巡查等场景。传统做法是打开多个浏览器窗口分别查看，窗口切换繁琐且消耗大量系统资源。

本项目旨在利用 C# WinForms + WebView2 技术，建立一个轻量级的直播并发控制系统，在单一窗口中嵌入多路直播画面，并引入并发控制机制（信号量）限制同时播放的直播流数量，防止系统资源耗尽。

## 2. 功能需求分析

### 2.1 核心功能

| 编号 | 功能 | 描述 |
|------|------|------|
| F1 | 多路直播嵌入 | 3 个直播间面板，每个可独立选择频道并开播/停播 |
| F2 | 并发数控制 | 通过信号量机制限制同时播放上限（默认 3），超出拒绝启动 |
| F3 | 频道选择 | 每个直播间下拉框选择 B站/抖音/虎牙/斗鱼等 6 个频道 |
| F4 | CSS 注入净化 | 自动隐藏直播页面的导航栏、弹幕、侧栏、广告等干扰元素 |
| F5 | 自适应缩放 | 根据面板大小自动计算 WebView2 ZoomFactor，适配小窗口展示 |
| F6 | 一键启停 | "全部开播" / "全部停播"按钮批量控制 |
| F7 | 实时状态栏 | 底部显示当前并发状态、活跃直播数、可用槽位数 |

### 2.2 用户场景

- **场景1**：运营人员打开 3 路不同平台的赛事直播，同时监控比赛进展
- **场景2**：某路直播卡顿，手动停播后启动其他频道的直播（槽位释放与重新分配）
- **场景3**：下班前点击"全部停播"，所有直播流立即停止，资源回收

## 3. 技术选型理由

| 技术 | 版本 | 选型理由 |
|------|------|---------|
| .NET Framework | 4.7.2 | 课程要求，WinForms 最稳定支持的运行时 |
| Windows Forms | — | 课程核心技术栈，适合桌面小工具快速开发 |
| WebView2 | 1.0.2903.40 | 基于 Chromium 内核，支持现代 Web 页面渲染，替代已废弃的 WebBrowser 控件 |
| C# | 7.3 | 与 .NET Framework 4.7.2 匹配的语言版本 |
| MSTest | 3.6.4 | 微软官方测试框架，与 Visual Studio 深度集成 |

### 为什么不用其他技术？

- **不用 WPF/WebView2 in WPF**：学习成本高，WinForms 更符合课程内容
- **不用 VLC/ffmpeg 直接拉流**：各平台直播地址非公开 RTMP/RTSP，WebView2 通过网页访问更通用
- **不用 Electron**：不符合课程 Windows 程序设计主题，且内存占用大

## 4. 系统架构设计

### 4.1 架构图（三层）

```
┌─────────────────────────────────────┐
│           Form1 (UI 层)              │
│  ┌───────┐ ┌───────┐ ┌───────┐     │
│  │Panel A│ │Panel B│ │Panel C│     │  ← GroupBox × 3
│  │WebView│ │WebView│ │WebView│     │
│  ├───────┤ ├───────┤ ├───────┤     │
│  │Combo  │ │Combo  │ │Combo  │     │  ← DropDown 选频道
│  │Button │ │Button │ │Button │     │  ← 独立启停按钮
│  └───────┘ └───────┘ └───────┘     │
│  [全部开播] [全部停播] [状态栏]      │
└────────────┬────────────────────────┘
             │ 依赖
┌────────────▼────────────────────────┐
│     LiveStreamManager (业务层)       │
│  - TryStart(id)  : 获取并发槽位     │
│  - Stop(id)      : 释放槽位         │
│  - StopAll()     : 全部停止         │
│  - IsRunning(id) : 状态查询         │
│  - ActiveCount   : 当前活跃数       │
│  - AvailableSlots: 可用槽位数       │
└─────────────────────────────────────┘
```

### 4.2 并发控制流程

```
用户点击"开播"
    │
    ▼
LiveStreamManager.TryStart(id)
    ├── 该流已在运行？ → 返回 false（防重复启动）
    ├── ActiveCount >= 3？ → 返回 false（槽位已满）
    └── 分配槽位，ActiveCount++ → 返回 true
        │
        ▼
    WebView2.Navigate(url) → CSS 注入 → 显示直播画面
        │
    Task.Delay(-1, ct) 等待直到...
        │
    用户点击"停播" → CancelToken → 停止直播
        │
        ▼
    LiveStreamManager.Stop(id) → ActiveCount--
```
